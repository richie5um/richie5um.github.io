Support/Self-Fix

When was the last time you contacted a Support team for help with an application?

I know for me it is very rarely.

When I encounter a problem with an application, the first thing I do, is to try and resolve the problem myself. If this fails, and the problem still exists, I’ll speak to other people (usually colleagues) to see if they have encountered it. And thirdly, I’ll scour the internet looking for an alternative product.  Very very rarely, even more so if the software hasn’t cost me anything, will I consider contacting the relevant Support team.

Why is this important?

Well, as you know, I’m a Software Developer/Engineer/Programmer.  And therefore, I’m responsible for producing products that may encounter the very same users as myself.  With this in mind, it becomes even more important to make sure the product is usable, especially when it is having problems - which sounds counter-intuitive.  Perhaps supporting all of the following charactistics would help:

  • Intuitive to use.
  • Intuitive to use, when having problems.
  • Obvious how to diagnose problems (a “.log” file in an obvious directory, with clear log messages).
  • Help is easy to launch.
  • Help presents an easy to use “Submit problem” .
  • Help is context sensitive.
  • Help is scenario based, rather than a disection of each individual UI component.

[There are many more, but these are the most obvious that currently spring to mind.]

Now, the self-fix approach probably differs when it is a business critical, and/or you’ve paid good money for it.  But I’d guess that even in these situations, a good proportion still attempt self-fix first.

This situation arose very recently for me.  I was using a particular Virtual Machine (Type2 Hypervisor) technology, and couldn’t get a Virtual Machine to boot without hanging.  I ran through my self-fix steps, and ended up wasting a decent amount of time before finally switching to another product - which worked first time.  Now, I’m fairly sure that the problem was something to do with my machine, as I’ve used that original product for a long time without any issues, and have only encounter this problem since changing laptops.  But, the end result was that I’ve lost faith in that product, and will be unlikely to try it again for a significant period of time.  And this is all without even contacting Support.

I’d be interest to hear if you are the same as me, or if you fully utilize Support.
