WYSIWYG - BNWYW

(What you see is what you get - but not what you want).

I remember the first time I heard of WYSIWYG; it was when I was at school, and my the IT teacher was mightly impressed with this new technology (and the wizzy name).  Step forward 15 (or so) years, and whilst the editors have many more features - tables, colour, media - are they actually closer to addressing the problems of creating content?

Content might be king, but, clear, consistent presentation is also very important.  Why is it then that achieving a consitent look and feel is so difficult in MS Word?  Maybe I’m missing something, but whenever I use it, I spend 50% of the time on writing content, and 50% of the time battling with the auto layout / custom formatting.  All I really want to do is markup my content as I go along, and worry about the presentation later.

CSS addresses many of these issues.  But, whilst it is an important technology, it is completely hampered by the divergent browser implementations.  Which with the battles IE8 and its standards compliant mode is having, this isn’t going to disappear any time soon.

Why is this important to me, well, in my role as a software engineer, it is vitally important to be able to capture and distribute understandable information, quickly and efficently.  This underpins not only software design, but also investigations, amongst many other tasks.

Maybe CSS and document writing will come together at some point in a useable (mainstream) product.  Either that or I’ll have to start using LaTeX.
